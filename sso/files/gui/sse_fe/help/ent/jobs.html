Jobs

Jobs are single state or orchestrate files that you upload to SaltStack Enterprise Web Console. Uploaded state and orchestrate files can reference files on your Salt Master, as well as other Salt States in the state tree.

Job files are distributed to the base environment on each Salt Master when the job is saved, and are then executed as needed the same as any other state or orchestrate file. All resources in your existing state tree and file servers are available.

State File Types

Minion-side state file

Lets you define a configuration to apply to one or more targeted Salt Minions. Minion-side state files are executed on all targeted Salt Minions simultaneously and the results are returned and stored in the database.

Deploying a Minion-side state file is the same as calling state.sls from the command line.

Master-side state file

Lets you define a series of commands and states each with their own targets. Master-side state files are executed on the Master.

Master-side state files should be used when servers need to be set up and configured in a specific order, and systems should be set up only if systems earlier in the sequence have been set up without any issues. Master-side state files can be used to deploy jobs in a smooth and reliable way across multiple systems in small to large environments.

Deploying a Master-side state file is the same as calling salt-run state.orchestrate from the command line.

Note:

Master-side state files are run by targeting a Minion that is attached to the Master on which you want the job to run. Even though a Minion is selected, the job runs on the Master.

Public and Private Jobs

Public Jobs are visible to all users that can log in to SaltStack Enterprise Web Console.

Private Jobs are visible to only you.

Job History

Job results are viewed by selecting one or more Minions and then clicking the Job History icon at the bottom of the Minion view:

Job History is displayed for the selected Minions.

State File Distribution

After a job is created, the state file is copied to the root of the base environment of each Salt Master. Note that state files uploaded to the Web Console are not renamed. You must ensure that file names you upload are unique and do not conflict with other state files that might be present in the state tree.

Note:

When adding a new Salt Master, any state files that were previously uploaded to the Web Console are not automatically copied to the new Master. You must manually copy these files to the new Master.
